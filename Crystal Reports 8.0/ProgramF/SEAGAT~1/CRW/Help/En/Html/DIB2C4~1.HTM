Reporting tab

The reporting options on this tab let you place a governor on data selection; they let you specify how you want Date/Time strings handled (if they are used in your report); and they let you specify whether or not data will be saved with your report.

Date-Time Field options

You have three options for converting Date-Time Fields:

  • Convert to String
  • Convert to Date
  • Keep Date-Time Type.

See also the Report Options dialog box.

Convert NULL Field Value to Default

Select this check box to have the program convert any null values to the database default. Some databases treat a null as a zero, some treat it as a blank, and some treat it as a special null value. This option is not selected by default. See also the Report Options dialog box.

COUNT Number of Records First

Select this check box (and then supply a maximum value in the Warn If More Than option) if you want to limit the number of records that can be included in a report. When you select the check box, the program will count the number of records first before building your report. By default, the check box is not selected.

Warn If More Than

Use this box to specify the maximum number of records the program can use in a report without first giving you a warning prompt. This option applies only to the Count Number of Records First check box.

Save Data With Report

When you select this check box, the program automatically saves database data with your report. Otherwise, the program saves the report itself, but it will need to retrieve the report data once again when you run the report. This option is selected by default. See also the Report Options dialog box.

Discard Saved Data When Loading Reports

Select this check box to discard saved data when loading reports. When you open a report, this function removes that report's current set of data from the database, so that you can enter new data while retaining the report's format and settings. This option is not selected by default.

Save Summaries With Report

Use this check box to determine whether the program will save the data summaries you create with the report. This option is selected by default. See also the Report Options dialog box.

Suppress Printing if No Records Selected

Use this check box to ensure reports cannot be printed if the database does not return any records. This option is not selected by default. See also the Report Options dialog box.

Show All Headers on Drill Down

Use this check box if you would like report drill-down views to included headers for all sections above the drill-down records. This option is not selected by default. See also the Report Options dialog box.

Autosave Reports After ... Minutes

Use this check box if you want reports to be saved automatically as you work on them. Use the Minutes box to specify the interval before and between autosaves. This option is not selected by default.

More Report Engine Error Messages

Select this check box to tell the program to offer more explanation in its error messages when a Report Engine error occurs. This option is selected by default. See also the Report Options dialog box.

Formula Language

Use this list to choose the formula language you want the program to use. You can choose between Crystal Syntax and Basic Syntax. Crystal Syntax is selected by default.
