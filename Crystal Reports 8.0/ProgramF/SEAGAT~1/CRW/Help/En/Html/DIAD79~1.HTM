Insert Subreport dialog box

This dialog box appears with the Subreport tab active when you choose the Subreport command from the Insert menu, or when you click the Insert Subreport button on the Supplementary toolbar.

Use the Insert Subreport dialog box to insert a subreport into your report. Once the dialog box appears, you can either choose an already existing report to insert as a subreport or create a new report from scratch.

Choose a report

When you choose this option, the Report File name box under the option is activated. Enter the name of the report you want to use as a subreport. If you do not know the name of the report, click Browse to find the file.

Report File name

Enter the full path for the report file you want to insert as a subreport. If you do not know the exact path click Browse to find the desired file.

Browse

Click the Browse button to find a report you'd like to include as a subreport. Double-click the report, or select Open and you are returned to the Insert Subreport dialog box. Notice that the file path to the report you just selected is now in the Report file name box.

Create a subreport

When you choose this option, the Report Name edit box under the option is activated. Enter a name for the subreport and then click the Report Expert button to create the new subreport.

Report Name

Enter the name you would like to assign to your new subreport and then click the Report Expert button to create your new report.

Report Expert

Click this button to open the Subreport Expert. Follow the directions on the tabs to create a new subreport.

On-demand subreport

Select this check box if you want your subreport to be On-demand. Data for an On-demand subreport is not read from the database until you drill down on the hyperlink used to represent it. Otherwise, subreport data is collected at the same time as the data for the container report.
