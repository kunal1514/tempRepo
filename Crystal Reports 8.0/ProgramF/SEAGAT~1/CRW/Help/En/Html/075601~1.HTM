Selection performance tips

Consider the following performance-related items when setting up record selection requests:

General

  • To push down record selection, "Use Indexes or Server for Speed" must be on.
  • Avoid date type conversion in record selection formulas.

    For example, using ToText ( ) to convert a numeric field to a string.

  • Able to push down some record selection formulas that use constant expressions.

PC Databases

  • Can only push down selection on indexed fields.
  • Able to push down AND clauses only (not OR).

SQL Databases

  • Able to push down selection on indexed or non-indexed fields.

    Note:    Server will respond faster if you use indexed fields.

  • Able to push down AND and OR clauses.
  • Record selection which contains some types of embedded formulas can be pushed down.
  • Use SQL Expression fields to push down formula calculations necessary for the record selection.

    For example, a record selection of {@ExtendedPrice > 1000} where @ExtendedPrice = (Orders x Quantity) can't be pushed down. However, if @ExtendedPrice is replaced with a SQL Expression field it will be pushed down.

  • Use Show SQL Query to view SQL that will be sent to server.
