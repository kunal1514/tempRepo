Browse Data dialog box

This dialog box appears when you click the Browse Data button in a dialog box or when you choose the Browse Data option from the shortcut menu on a selected field.

You can browse data for example:

  • from the Choose Field dialog box
  • in the Select Expert
  • from the Field Explorer.

Note:    From the Field Explorer, you can browse database fields, formula and SQL expression fields.

Use the Browse Data dialog box to preview the data type, length, and content of any field.

  • The source of the field (alias) and the field name appear as the title for the dialog box.
  • The data type for the field (string, number, and so on) appears just below the title.
  • The list contains a list of field values.

Note:    When you browse a field, by default you see the first 500 unique values of the field. You can change this for the current report by clearing the Select Distinct Data for Browsing option in the Report Options dialog box. You can change the default setting by clearing the Select Distinct Data for Browsing option on the Database tab of the Options dialog box.

If Select Distinct Data for Browsing is not selected, browsing returns the unique values in the first 500 records.

Note:    Note that since most SQL servers do not support selecting distinct records for long data types such as memo and blob fields, selecting and browsing distinct records are not supported for such fields.

If you fail to connect to a database because it is unavailable or password-protected, you can still browse on saved data.
