SQL Expression Fields

Use the SQL Expression Fields branch of the tree as a gateway for editing, renaming, or deleting SQL Expression fields, creating new SQL Expression fields, or inserting SQL Expression fields into your report.

The SQL Expression Fields list displays the names of all the expression fields currently in your report. You insert SQL Expression fields by dragging them into the report, or by selecting them and then clicking Insert to Report.

Each SQL Expression field added to the report has a check mark added in front of its name in the Field Explorer.

Insert to Report

Use this button to insert the SQL Expression field selected in the SQL Expression Fields list into your report.

New

This button opens the SQL Expression Name dialog box where you can specify a name for your expression. This option is available from the root of the SQL Expression Fields branch so you can create a new SQL Expression for your report even when there are no existing expressions.

To add a new SQL Expression, add a name in the SQL Expression Name dialog box, click OK, and create a expression in the SQL Expression Editor.

Edit

This button opens the SQL Expression Editor and displays the currently selected expression for you to edit.

Rename

This button lets you rename the expression you select in the SQL Expression Fields list. SQL Expression Fields can be renamed in the standard Windows fashion.

Delete

This button deletes the expression field selected in the SQL Expression Fields list. If the expression is already on the report, you are asked to confirm the deletion as it will also be deleted from the report.
