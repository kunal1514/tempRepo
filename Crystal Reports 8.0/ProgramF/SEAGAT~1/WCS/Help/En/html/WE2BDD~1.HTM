Data source considerations

If the data source requires permissions that are greater than those of the local system account you will not be able to access the data file and you will receive looping logons. Two examples of this are outlined below:

  • If your reports are created using a PC database (Microsoft Access or Microsoft FoxPro) and you reference the data file with a UNC path, or ODBC DSN is referencing a UNC path, and the Page Server is running under the local system (or any other local account).
  • If your reports access the data through a database client like Informix Metacube or Arbor Essbase, and the Page Server is running under the local system (or any other local account).

To correct this, change the account the Page Server uses to a domain account that has access to the network share where the data is stored.
