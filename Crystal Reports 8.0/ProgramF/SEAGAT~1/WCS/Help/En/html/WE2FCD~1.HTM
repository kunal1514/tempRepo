Configuring Lotus Domino to work with the Web Component Server

If you are using a Lotus Domino server, complete the following procedure before accessing reports over the Web.

  1. Copy the wcscgi.exe file into the Lotus Domino cgi-bin directory.

    This gives Domino Webserver access to the Crystal Reports Web Server program files through its CGI directory.

  2. Create a virtual directory in Lotus Domino for Seagate Crystal Reports.

    Use the Domino Configuration database to do this. If you do not have a Domino Configuration database, create one called domcfg.nsf in the Domino Server default data directory using the Domino Configuration template (domcfg.ntf).

  3. Map the URLs to the report.

    The steps for doing this will vary depending on the version of Domino you are using. Refer to Domino's online help for more information.

      • Mapping Directories in Lotus Domino Version 4.x

    Using the Domino Configuration Database follow the path Create -> Mapping URL -> Directory and map the four directories below:

    URL Path Directory

    1

    /scrreports/*

    c:\Progra~1\Seagat~1\Crysta~1\Reports\*

    2

    /scrsamples/*

    c:\Progra~1\Seagat~1\Crysta~1\samples\ code\web\*

    3

    /viewer/*

    c:\Progra~1\Seagat~1\Viewers\*

    4

    /reports

    d:\notes\data\report

    The first two mappings are optional. They enable you to view samples that are included with the product.

    The third mapping is required; this is the directory where the viewer clients are stored. When a browser requests a report, the Web Component Server interprets the browser and automatically loads the proper viewer (Java, ActiveX, HTML, or HTML with Frames).

    The fourth mapping is required. This sets the current directory on the server whenever a request comes in for a report. The actual name of the directory (in this case /reports) is whatever you set up under the Notes data directory. It is activated via the URL redirection below.

    In Directory Mapping, you can specify the IP address to which the mapping applies. This is only necessary if you are running multiple Virtual Servers and wish to restrict reporting access to one of them. All of the Directory Mappings should also be set to Read access not Execute. It is recommended to enter something in the comment field for reference, but this is optional.

  4. Restart the Lotus Domino server.
  5. Ensure NT Services have started.
      • Choose Services from the Control Panel.

        The Services dialog box appears.

      • Confirm the Seagate Page Server and Seagate Web Component Server services have started. If they are not running, click Start.

  6. Access sample reports by entering the URL into your web browser

    http:/localhost/scrsamples
